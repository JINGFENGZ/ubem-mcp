                    GitHub 快速部署指南
================================================================================

方式1: 使用自动化脚本（推荐）
================================================================================

【Windows用户】

1. 打开PowerShell，进入项目目录：
   cd D:\Files\LLM-RAG-UBEM_New\ubem-mcp-opensource

2. 运行部署脚本（替换YOUR_USERNAME为你的GitHub用户名）：
   .\DEPLOY_TO_GITHUB.ps1 -GithubUsername YOUR_USERNAME

   例如：
   .\DEPLOY_TO_GITHUB.ps1 -GithubUsername zhangsan

   或自定义仓库名：
   .\DEPLOY_TO_GITHUB.ps1 -GithubUsername zhangsan -RepoName my-ubem-mcp

3. 脚本会自动完成：
   - 更新所有文件中的YOUR_USERNAME
   - 初始化Git仓库
   - 添加所有文件
   - 创建初始提交
   - 设置远程仓库

4. 然后按照脚本提示：
   a. 在GitHub上创建新仓库: https://github.com/new
      - 仓库名：ubem-mcp（或你指定的名称）
      - 可见性：Public
      - 不要初始化README
   
   b. 返回PowerShell，推送代码：
      git push -u origin main

【Linux/Mac用户】

1. 给脚本添加执行权限：
   chmod +x DEPLOY_TO_GITHUB.sh

2. 运行脚本：
   ./DEPLOY_TO_GITHUB.sh YOUR_USERNAME

   例如：
   ./DEPLOY_TO_GITHUB.sh zhangsan ubem-mcp

3. 按照脚本提示完成后续步骤

================================================================================
方式2: 手动部署
================================================================================

【步骤1：更新文件】

手动编辑以下文件，将YOUR_USERNAME替换为你的GitHub用户名：
- README.md
- CHANGELOG.md
- pyproject.toml
- CONTRIBUTING.md

【步骤2：初始化Git】

cd D:\Files\LLM-RAG-UBEM_New\ubem-mcp-opensource
git init
git add .
git commit -m "Initial commit: UBEM Analysis MCP Server v1.0.0"
git branch -M main

【步骤3：在GitHub创建仓库】

1. 访问：https://github.com/new
2. 仓库名：ubem-mcp
3. 描述：Urban Building Energy Model Analysis MCP Server
4. 可见性：Public
5. 不要初始化README、.gitignore或LICENSE（我们已经有了）
6. 点击"Create repository"

【步骤4：推送代码】

git remote add origin https://github.com/YOUR_USERNAME/ubem-mcp.git
git push -u origin main

================================================================================
认证方式
================================================================================

【如果启用了2FA（双因素认证）】

你需要使用Personal Access Token（个人访问令牌）：

1. 创建Token：
   - 访问：https://github.com/settings/tokens
   - 点击"Generate new token (classic)"
   - 勾选"repo"权限
   - 生成并复制token

2. 推送时使用token：
   当git push要求输入密码时，输入token（不是你的密码）

【使用SSH（推荐）】

如果你配置了SSH密钥：
git remote set-url origin git@github.com:YOUR_USERNAME/ubem-mcp.git
git push -u origin main

================================================================================
创建Release（可选）
================================================================================

推送代码后，创建第一个正式发布版本：

1. 访问：https://github.com/YOUR_USERNAME/ubem-mcp/releases/new

2. 填写信息：
   - Tag: v1.0.0
   - Release title: UBEM Analysis MCP Server v1.0.0
   - Description: 复制CHANGELOG.md中的内容

3. 点击"Publish release"

================================================================================
发布到PyPI（可选）
================================================================================

如果你想让别人通过pip安装你的工具：

1. 注册PyPI账号：https://pypi.org/account/register/

2. 安装构建工具：
   pip install build twine

3. 构建包：
   python -m build

4. 上传到PyPI：
   twine upload dist/*

5. 然后别人就可以安装了：
   pip install ubem-analysis-mcp

================================================================================
常见问题
================================================================================

Q: 推送时提示"Permission denied"
A: 检查你的GitHub认证。使用token或配置SSH密钥。

Q: 推送时提示"Repository not found"
A: 确保在GitHub上已创建仓库，且仓库名匹配。

Q: 想要修改仓库名
A: 在脚本中使用-RepoName参数，或手动修改remote url。

Q: 已经push过，想要重新开始
A: 删除.git文件夹，重新运行脚本。

Q: 脚本执行被阻止（Windows）
A: 运行: Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
